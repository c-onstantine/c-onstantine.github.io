github

issues板块，相当于一个版本追踪，可以在这里查看一些用户提出的问题，和作者的解决方法，这个板块 也是判断项目是否活跃的一个标准。

pull requests板块，发出提交贡献，等待项目作者审查 是否何如你的代码。 pr

projects板块 项目管理工具 ，可以做一个工作流程表 ，工作计划 ，需求列表 ；（可以列一个计划）

wiki 板块 项目的知识库 介绍

insights项目的一些基本信息

settings板块项目的设置

  • webhooks 消息触发机制 例如 做一个代码审查 每次提交后 通过webhooks给代码审查提交信息，从而运行代码审查程序，从而实现 消息的传递
